Critical Findings

5 Must-Fix Before Any Deployment

These findings represent active security vulnerabilities that could be exploited by any authenticated user.

CRIT-1
CSRF Bypass — Any Bearer Header Skips Protection
The CSRF middleware skips validation for any request with Authorization: Bearer ..., even a fake token. The Collection frontend sends zero CSRF tokens. Combined: any attacker with a session cookie can bypass CSRF entirely.
middleware/csrf.py:27
CRIT-2
OTP Rate Limiter Completely Broken
Rate limit never triggers because OTP records are deleted before counting. Each request deletes the previous record, so the count check always finds 0-1 records, never reaching the threshold of 5. Unlimited OTP brute-force possible.
routers/auth.py:101
CRIT-3
Cross-Tenant Data Leak — Health Alerts Map
/v1/health/alerts/map returns ALL health events with district/village locations to any authenticated user. A farmer sees every other farmer's livestock health and location data. No role check, no user scoping.
routers/health.py:131
CRIT-4
Milk Center Endpoints — No Role Checks
All 5 milk center endpoints (receive, daily-report, settlements, farmers/search, farmers/enroll) have no role validation. Any farmer can record milk for others, search ALL farmer PII (phone, Aadhaar), and enroll new accounts.
routers/milk_center.py
CRIT-5
Financial Calculations Use Float, Not Decimal
Transaction amounts, milk pricing, and settlement calculations use Python float. Cumulative rounding errors across hundreds of INR transactions will produce incorrect settlement totals. DB columns are Numeric(10,2) but input/aggregation uses float.
routers/finance.py:26
